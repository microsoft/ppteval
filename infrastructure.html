PPT-Eval runs tasks inside a sandboxed PowerPoint Online session. Agents issue GUI-level actions (selection, typing, dragging, focus changes) mirroring human interaction and revealing realistic failure modes like mis-clicks or unintended global style changes.

Execution Loop. Each task specifies a natural language goal; the agent iteratively observes updated state (DOM/screenshot/metadata) and selects the next action until completion or a budget of 30 steps is exhausted. We use a concurrency of 3 threads (~3.5 hours per full run).

Rubric Scoring. Each task is represented by a tree-structured rubric where internal nodes define higher-level criteria and leaf nodes implement concrete checks. Critical nodes correspond to core requirements; non-critical nodes capture desirable but secondary aspects (e.g. stylistic choices, penalizing extraneous changes). Each leaf returns a numeric score (∈[0,1]) and a natural language explanation.

Aggregation Strategy. Modified scoring combines average critical progress with a penalty factor (λ=0.3) from non-critical shortcomings, avoiding collapse of near-complete attempts to zero while preserving importance of essential changes.

Human & Model Collaboration. Rubrics are generated semi-automatically: LLMs (Claude-4-Sonnet and GPT-4.1) propose draft rubric trees, which are then refined via multi-stage expert review by six human annotators to ensure consistency and robustness.

Meta-Evaluation. Rubric-based scores show strong agreement with human judgments: Kendall's τb = 0.77, Spearman's ρ = 0.84.
